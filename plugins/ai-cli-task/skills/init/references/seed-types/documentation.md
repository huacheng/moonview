# documentation

## Description

Docs, README, translation, content

## Methodology

Prose linting, link validation, reading level

## Phase Intelligence

### plan

- **Collection Direction**: Content architecture, style guides, terminology standards, audience analysis
- **Key Sources**: Google/Microsoft style guides, Divio documentation system, i18n best practices, accessibility (WCAG)
- **Plan Structure**: Outline → content structure → cross-references → review criteria
- **Key Considerations**: Completeness, link validity, terminology consistency, audience appropriateness

### verify

- **Collection Direction**: Link checkers, spell checkers, readability analysis, accessibility testing
- **Key Sources**: Lychee, cspell, Vale, Hemingway, axe-core, markdownlint
- **Quick Checkpoint**: Markdown renders, no broken links
- **Full Checkpoint**: Completeness checklist pass, spell check, terminology consistency, link validation, formatting rules
- **Key Tools**: `markdownlint`, `cspell`, `lychee` (link checker), `vale`, `textlint`

### check

- **Collection Direction**: Documentation quality standards, completeness criteria
- **Key Sources**: Divio quality criteria, API documentation best practices, accessibility standards
- **Indicators**: Docs, README, translation, writing
- **Verification Approach**: Completeness checklist, link validation, spell check, formatting consistency

### exec

- **Collection Direction**: Markup syntax, template systems, publishing tool chains
- **Key Sources**: Markdown/AsciiDoc specs, static site generator docs, PDF/EPUB toolchain guides
- **Implementation Approach**: Write/edit content, validate links, check formatting
- **Step Verification**: Completeness checklist, link validation, spell check
